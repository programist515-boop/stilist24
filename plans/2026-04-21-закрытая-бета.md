### Критичные проблемы

1. **Pytest не запускается: 26 collection errors.** Все тесты падают с `ModuleNotFoundError: No module named 'app'`.
   - **Причина:** в [ai-stylist-starter/Dockerfile:14-17](../ai-stylist-starter/Dockerfile#L14-L17) команда `pip install .` выполняется **до** `COPY . /app`. На момент установки в `/app` есть только `pyproject.toml`, сам пакет ещё не скопирован. Setuptools ставит в `site-packages` только `ai_stylist_starter-0.1.0.dist-info` (метаданные), без кода.
   - API работает, потому что `uvicorn app.main:app` запускается с `cwd=/app`, и Python добавляет cwd в `sys.path`.
   - Pytest добавляет в sys.path `/app/tests`, а не `/app` — поэтому тесты не видят модуль `app`.
   - **Минимальный фикс:** добавить `[tool.pytest.ini_options] pythonpath = ["."]` в `pyproject.toml` ИЛИ положить `conftest.py` в корень с `sys.path.insert`.
   - **Правильный фикс:** в Dockerfile поменять порядок — `COPY . /app` до `pip install -e .` (editable mode, чтобы код из volume реально работал).

2. **`alembic current` падает** с тем же `No module named 'app'`, если запускать не из `cwd=/app`. Фикс тот же — правильный PYTHONPATH.
