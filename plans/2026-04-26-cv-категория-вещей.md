# Точное автоопределение категории при загрузке вещи в гардероб

> **Зеркало:** канонический план — `~/.claude/plans/wise-strolling-beaver.md`. Здесь хранится копия для соответствия правилам проекта (`CLAUDE.md` → «План для каждой новой функции»).

## Context

Сейчас при загрузке фото в гардероб ([wardrobe.py:149](../ai-stylist-starter/app/api/routes/wardrobe.py)) категория = `category or "top"`. ML-классификатор не реализован — это закрытый честный долг из саммари 2026-04-26. Workaround — ручной дропдаун «Не та?» в [WardrobeUploader.tsx:184-196](../frontend/src/components/wardrobe/WardrobeUploader.tsx).

**Почему сейчас:** перед закрытой бетой (30 тестеров через 1-2 недели) хотим минимизировать ручной труд при онбординге. Параллельно — расширить категории с 6 плоских (top/bottom/...) до 15, как уже описано в [config/rules/category_rules/](../ai-stylist-starter/config/rules/category_rules) — для более точного скоринга.

**Согласованные решения:**
- Подход: **Claude Haiku 4.5 Vision** через прокси **kie.ai** (Anthropic-compatible API)
- Категории: **15** (bags, belts, blouses, dresses, eyewear, headwear, hosiery, jackets, jewelry, outerwear, pants, shoes, skirts, sweaters, swimwear)
- Rollout: **merge в main с флагом OFF**

**Уточнения по ходу реализации:**
- kie.ai endpoint: `POST https://api.kie.ai/claude/v1/messages` с `Authorization: Bearer`. Anthropic-compatible body, имя модели `claude-haiku-4-5` (без даты).
- **kie.ai не поддерживает forced `tool_use` / `tool_choice`** — при наличии этих полей возвращается `content: []` и `output_tokens=0`. Поэтому используется **JSON-in-text** через сильный system prompt + парсер с поддержкой code-fence (```` ```json ... ``` ````) и embedded prose.
- HTTP-клиент: **httpx напрямую**, без `anthropic` SDK — даёт контроль над `base_url` и `auth_scheme` (`bearer` для kie.ai, `x-api-key` для прямого Anthropic). Drop-in замена провайдера через одно env-переключение.
- Ключ от kie.ai в `.env` (gitignored, корневой `.gitignore` строка 30) для локального теста; в проде — через GitHub secret `CLAUDE_API_KEY`.

## Архитектура

```
POST /wardrobe/upload (image, category=None)
  → recognize_garment(image, hint_category=None)
  → if category is None and USE_CV_CATEGORY_CLASSIFIER:
       pred = classifier.classify(image, attrs_hint=detected)
       resolved_category = pred.category if pred.confidence >= 0.6 else None
       category_meta = {value, confidence, source}
     else:
       resolved_category = category or None
  → repo.create(category=resolved_category, attributes={...meta})
```

При confidence < 0.6 сохраняем `category=None` — фронт показывает «Похоже на X — уточни?».
