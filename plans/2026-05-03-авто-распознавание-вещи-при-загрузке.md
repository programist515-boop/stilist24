# Авто-распознавание вещи при загрузке в гардероб

Дата: 2026-05-03.
Источник: `~/.claude/plans/happy-juggling-pie.md` (утверждённый план чата 2026-05-03).
Связанный rollout: за пользователем — настройка GitHub Secrets `OPENAI_API_KEY` + `OPENAI_HTTP_PROXY` через proxy-seller.com.

## Context

Загрузка вещи в гардеробе была трёхступенчатой: пользователь выбирал файл, опционально выбирал категорию из dropdown («Угадаем по фото»), жал «Загрузить вещь». Бэкенд гонял эвристики `recognize_extended()` (8/14 атрибутов из Phase 0 покрыто) и опционально gpt-5-nano для категории. Имени вещи не было.

Цель: один vision-вызов через **GPT-5 mini** возвращает категорию, имя, цвет и 14 структурных атрибутов разом. Фронт автоматически шлёт upload по выбору файла, без кнопки. После ответа пользователь видит редактируемые поля и кнопку «ОК».

## Решения по развилкам

- **Background removal:** rembg + u2net (уже стоит, не меняем).
- **Vision провайдер:** GPT-5 mini через `OPENAI_BASE_URL=https://api.openai.com` + `OPENAI_HTTP_PROXY` на западный HTTP-прокси (proxy-seller.com / ISP Proxies). RU-IP блокировка обходится сетевым прокси, не менять endpoint.
- **Имя вещи:** новая колонка `wardrobe_items.name TEXT NULL` (alembic 0011), не в `attributes_json` — поле user-facing, должно быть searchable.
- **Pre-upload Select категории:** убран. Категория всегда определяется vision'ом, правится только в карточке после ответа.
- **Подтверждение:** явная кнопка «ОК» с diff-PATCH'ом, без auto-save на blur.
- **Прогресс:** псевдо-stepper, переключение по `setInterval(2500ms)` в фоне одного `mutation.isPending`. Под капотом цепочка реально идёт «vision → save nobg → DB», подписи не лгут.

## Фазы

### Фаза 1. Backend [x]

#### 1.1. `OpenAIVisionAnalyzer` в `category_classifier.py` [x]

- `VisionAnalysisResult(category, confidence, name, primary_color, attrs, source, reasoning)` dataclass.
- Новый класс `OpenAIVisionAnalyzer` делит инфру с `OpenAICategoryClassifier`: один httpx-клиент, breaker, retry, downscale до 1024px JPEG.
- `_VISION_SYSTEM_PROMPT` собирается из whitelist'ов в `app/models/item_attributes.py` — единый источник истины. При добавлении нового значения в whitelist промпт автоматически расскажет о нём модели.
- `analyze(image_bytes) -> VisionAnalysisResult` — один POST, парсит расширенный JSON. Невалидные значения (вне whitelist) → None через `validate_scalar` / `validate_style_tags`.
- `_coerce_name` урезает name до 60 символов по последнему пробелу.
- `get_vision_analyzer(settings)` фабрика — None при `enable_vision_analysis=False` или пустом `openai_api_key`.

#### 1.2. Миграция 0011 [x]

[`alembic/versions/0011_wardrobe_item_name.py`](../ai-stylist-starter/alembic/versions/0011_wardrobe_item_name.py): nullable `Text`-колонка `wardrobe_items.name`. В [`app/models/wardrobe_item.py`](../ai-stylist-starter/app/models/wardrobe_item.py) — `Mapped[str | None]`.

#### 1.3. Pipeline `routes/wardrobe.py` [x]

Параллельный CV-pipeline через `asyncio.to_thread`:
1. S3 upload оригинала
2. **Параллельно**: rembg-маска (`recognize_garment`) и vision-вызов (`analyzer.analyze`)
3. nobg PNG → S3 (детерминированный ключ `{key}_nobg.png`)
4. Если vision успешен — берём category, name, primary_color, attrs из его ответа. Иначе fallback на эвристики.
5. `repo.create(name=..., category=..., attributes=v2_dict, structured_attrs=14_attrs_dict)`

Latency сократилась ≈ в 1.5× (rembg ~2-4с + vision ~3-5с теперь идут параллельно вместо sequential).

#### 1.4. PATCH `/wardrobe/{id}` [x]

- Новый `WardrobeItemPatchIn(category?: str, name?: str)` в [`schemas/wardrobe.py`](../ai-stylist-starter/app/schemas/wardrobe.py)
- Новый endpoint `PATCH /wardrobe/{item_id}` принимает оба поля опционально, отправляет diff. Если оба None — no-op без записи.
- Старый `PATCH /wardrobe/{item_id}/category` сохранён для совместимости.
- `WardrobeRepository.create` расширен `name` и `structured_attrs` (плоский dict из 14 Phase-0 атрибутов, валидация на уровне SQLAlchemy `@validates`).

#### 1.5. Settings и env [x]

[`app/core/config.py`](../ai-stylist-starter/app/core/config.py):
- `enable_vision_analysis: bool = False` — отдельный от `use_cv_category_classifier` флаг
- `openai_model: str = "gpt-5-mini"` (был `gpt-5-nano`)
- `openai_http_proxy: str = ""` — сетевой прокси для исходящих vision-вызовов

[`.env.prod.example`](../.env.prod.example) и [`.github/workflows/deploy.yml`](../.github/workflows/deploy.yml) here-doc обновлены: новые env-переменные, новый Secret `OPENAI_HTTP_PROXY`.

#### 1.6. Тесты [x]

[`tests/test_category_classifier.py`](../ai-stylist-starter/tests/test_category_classifier.py): +14 кейсов:
- Vision возвращает все поля (category + name + primary_color + 14 attrs)
- Невалидные значения вне whitelist стираются в None
- `style_tags` фильтруется по STYLE_TAG_VALUES
- Long name обрезается до 60 символов
- `null` для name/color обрабатывается без падения
- `max_completion_tokens` (не `max_tokens`), `response_format: json_object` — guard'ы что код не регрессирует на старый API
- Unparseable response / unknown category → RuntimeError
- Factory: возвращает None при выключенном флаге / без ключа
- Прокси прокидывается через factory в analyzer и classifier
- Пустая строка proxy → None в httpx.Client

## Возможные проблемы при rollout

| Симптом | Что проверить |
|---|---|
| 403 в логах api | прокси-IP в РФ — нужен западный (DE/NL/PL/US) |
| Connection timeout | прокси упал, проверить активность плана в proxy-seller |
| 401 Unauthorized | `OPENAI_API_KEY` невалидный или без прав на vision-модели |
| Имя пустое, категория есть | vision упал, fallback на эвристики сработал — посмотреть `docker logs api` |
| `httpx.UnsupportedProtocol: socks5` | используешь SOCKS5 → поставить `httpx[socks]` или взять HTTP-прокси |

## Что переносится в следующие планы

Ничего критичного. Возможные follow-ups когда фича заработает на проде:
- Калибровка vision-промпта на реальных пользовательских фото (если accuracy < 80%)
- Few-shot examples в prompt для сложных категорий (jewelry vs accessories, swimwear vs blouses)
- Метрика `vision_correction_rate` (сколько юзеров правят категорию или имя через «ОК»-форму)
