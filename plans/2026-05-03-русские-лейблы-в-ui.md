# Русские лейблы для слотов, категорий и значений атрибутов в UI

Дата: 2026-05-03.
Связанная история: будет дописана в `.business/история/2026-05-03-русские-лейблы.md` отдельно.

## Context

На странице `/reference-looks` пользователь видел английские enum-ключи: `wrap_dress`, `kitten_heels`, `flowing/curved`, `midi/just_below_knee`, `accessory:bag`, `smart_casual`, `top`, `bottom`, `shoes`, `medium`, `soft` и т.д. Это data-ключи из YAML (`config/rules/reference_looks/*.yaml`, `category_rules/*.yaml`), которые до сих пор не переводились в UI. То же касается `/style-quiz` (шаг wardrobe-match) и `/recommendations` (через `gap_analysis`).

Решение: перевести **значения** на бэке в `_build_shopping_hint` (там уже переводились ключи атрибутов — расширяем) + **slot/occasion/category** на фронте для полей, которые приходят как стабильные ключи (`MatchedItem.slot`, `Look.occasion`, и т.п.).

## Фазы

### Фаза 1. Backend — расширить `_build_shopping_hint` [x]

В [`ai-stylist-starter/app/services/reference_matcher.py`](../ai-stylist-starter/app/services/reference_matcher.py):

- `_CATEGORY_LABELS_RU` (~80 ключей): top/bottom/dress/shoes + sub_types из YAML — `wrap_dress`, `kitten_heels`, `ballet_flats`, `ankle_boots`, `circle_skirt`, `midi_skirt`, `blouse`, `ruffled_top`, `silk_shell`, `loafers`, `oxfords`, `combat_boots`, `chunky_sneakers`, `coat`, `trench`, `leather_jacket`, `accessory:{bag,jewelry,belt,scarf,hat}` и т.д.
- `_VALUE_LABELS_RU` (~50 значений): cut_lines (`angular`, `straight`, `flowing`, `curved`), structure (`structured`, `semi_structured_curved`), pattern_scale/detail_scale (`small`, `medium`, `large`), pattern_character (`stripe`, `dots`, `geometric`, `floral_*`), neckline_type, sleeve_length, length (`midi`, `just_below_knee`, `ankle`), occasion (`day`, `work`, `smart_casual`, `evening`).
- `_humanize` (`foo_bar` → `foo bar`) как fallback для непокрытых ключей — UI больше не показывает подчёркиваний.
- `_build_shopping_hint` применяет `_localize_category` к `requires["category"]` и `_localize_value` к `expected`.

Это чинит `shopping_hint` сразу для всех потребителей — `/reference-looks`, `/style-quiz/wardrobe-match`, `/recommendations` (через `gap_analysis_service.py:277`).

### Фаза 2. Тест — обновить под RU-строки [x]

[`ai-stylist-starter/tests/test_reference_matcher.py:114-124`](../ai-stylist-starter/tests/test_reference_matcher.py#L114-L124): `test_shopping_hint_has_category_and_attrs` проверял подстроки `"heels"`/`"pointed_flats"` — переписан на русские: `"каблук"`/`"лодочки"`/`"балетки"`/`"ботильон"`.

### Фаза 3. Frontend — i18n словари [x]

Новый файл [`frontend/src/lib/i18n/wardrobe.ts`](../frontend/src/lib/i18n/wardrobe.ts) с тремя словарями и форматтерами:

- `SLOT_LABEL` + `formatSlot` — top/bottom/dress/shoes/outerwear/headwear + составные `accessory:bag` и т.п.
- `OCCASION_LABEL` + `formatOccasion` — day/work/smart_casual/evening + season hints (`all`, `spring_autumn`, ...).
- `CATEGORY_LABEL` + `formatCategory` — продублирован минимум того, что лежит в backend `_CATEGORY_LABELS_RU` (для UI-полей `MatchedItem.category`, которое приходит как сырой enum).

Fallback `humanize` (`foo_bar` → `Foo bar`) — никогда не возвращает пустую строку.

## Открытые вопросы

- `cut_lines: flowing/curved` → «плавные/плавные» — оба термина в YAML присутствуют как разные ключи, но в RU перевелись одинаково. Если стилистически важна разница — добавить отдельный термин для `curved` (например, «изогнутые»).
- Если в YAML появятся новые подтипы / категории — их перевод нужно добавлять в оба словаря (backend `_CATEGORY_LABELS_RU` и frontend `CATEGORY_LABEL`). Fallback `_humanize` сейчас даст хотя бы читаемые латинские слова без подчёркиваний.
